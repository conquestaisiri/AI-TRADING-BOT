Major problem 4: swing high / swing low logic is vulnerable to self-reference issues



This one matters.



The code computes:



rolling max of high

rolling min of low



Then the strategy checks whether the current close is above the current row’s swing_high or below swing_low.



Depending on how that rolling window is used, this can be logically weak because the current candle may already be part of the rolling calculation.



That can distort breakout detection.



Why this matters



A real breakout check usually compares current close to:



prior swing high excluding current candle

or

prior N-bar high shifted by 1



Same for lows.



If the rolling high includes the current candle’s own high, the breakout logic becomes less clean.



That is a serious detail.



This is exactly the kind of thing that makes a strategy look valid but behave weirdly.



So I would want the swing levels shifted:



previous swing high

previous swing low



Not same-row rolling values blindly used.
